# Team Documentation Writing Skill

Create team collaboration documentation optimized for internal workflows, process integration, and knowledge sharing. Primary platform: Notion.

## Focus Areas

### 1. Team Knowledge Bases
- Process documentation
- Team standards and conventions
- Onboarding materials
- Reference guides
- Decision logs

### 2. Collaborative Documentation
- Meeting notes with action items
- Project status updates
- Team dashboards
- Runbooks and SOPs
- Shared troubleshooting guides

### 3. Notion-Specific Features
- Database properties and relations
- Templates for consistency
- Views and filters
- Inline mentions and links
- Embedded content

## Structure Principles

- **Findable**: Clear navigation, searchable
- **Actionable**: Clear next steps, owners
- **Current**: Regular updates, version history
- **Collaborative**: Multiple contributors, comments
- **Integrated**: Links to tools (Jira, GitHub, Slack)

## Writing Style

- **Clear and concise**: Busy teams scan quickly
- **Action-oriented**: Focus on what to do
- **Consistent format**: Use templates
- **Cross-referenced**: Link related docs
- **Status-aware**: Indicate draft/review/approved

## Quality Checklist

- ✓ Clear ownership and maintenance plan
- ✓ Integrated with team tools
- ✓ Action items identified
- ✓ Cross-references to related docs
- ✓ Appropriate Notion formatting
- ✓ Regular update schedule noted

## Integration

Invoked by:
- `/doc write --audience=team` command
- Team documentation workflows
- Notion publishing pipelines
