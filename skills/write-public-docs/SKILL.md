# Public Documentation Writing Skill

Create polished, professional documentation for external audiences. Focus on completeness, professionalism, and positive first impressions.

## Document Types

### 1. Open Source Documentation
- Professional README with badges and examples
- Contributing guidelines (CONTRIBUTING.md)
- Code of Conduct
- Comprehensive API documentation
- Changelog and release notes
- License information

### 2. Client Deliverables
- Executive summaries
- Technical specifications
- Deployment guides
- Training materials
- Support documentation

### 3. Public API Documentation
- API reference (endpoints, parameters, responses)
- Authentication guides
- Rate limits and best practices
- SDK documentation
- Interactive examples

## Quality Standards

### Professional Polish
- Consistent formatting and style
- No typos or grammatical errors
- Professional tone throughout
- Complete information (no TODOs or placeholders)
- Working examples and code samples

### Comprehensive Coverage
- All features documented
- Edge cases explained
- Error handling covered
- Security considerations noted
- Performance characteristics documented

### Accessible Presentation
- Clear navigation
- Searchable content
- Multiple entry points (quickstart, reference, tutorials)
- Visual aids where helpful
- Mobile-friendly (for web docs)

## Writing Style

- **Professional but approachable**: Friendly without being casual
- **Precise**: Accurate technical information
- **Complete**: Answer all questions users might have
- **Tested**: All examples work as shown
- **Current**: Reflects latest version

## Quality Checklist

- ✓ No placeholders or TODOs
- ✓ All code examples tested and working
- ✓ Consistent formatting throughout
- ✓ Professional tone maintained
- ✓ Complete feature coverage
- ✓ Security and legal considerations addressed
- ✓ Contact/support information provided
- ✓ License clearly stated
